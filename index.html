RocketBlast

High-Level Description

The idea of this game is to have a rocket which gives the user a view of the inside of the cockpit. This will give the feel that the user is driving the rocket.

We want to have clouds moving towards the rocket, to give a feel that the user is moving forward, unfortunately, we currently have the clouds scrolling downwards!

We also plan on having objects move towards the cockpit, and have the user, shoot these objects to gain points. In the case that the user misses the object, they may face some sort of a consequence.

Currently, we've got the clouds scrolled downwards and an image of a cockpit being displayed. We would like to look at how we can make these clouds move towards the screen, rather than downwards. We would also like to upgrade our image of the cockpit at some point in time.

We have three projects which we auto-generated using libgdx's nightly package. For the time being, we decided to eliminate the browser package. Our three packages are as follows:

RocketBlast

This project consists one package. The package is as follows:

com.rocket.blast

This package consists of four files. These four files are as follows:

1. RocketBlast.java

This file contains all the main work for our game to function. This class will call upon RocketFlux, SkyFlux and AsteroidFlux in order to allow those classes to handle the work related directly to those objects. This allows for us to have a nicely abstracted way of handling our development and organization of the game.

2. RocketFlux.java

As mentioned earlier, this file will contain any and all work directly related to the rocket displayed on the screen. For instance, we plan on adding a joystick which will move as the user re-directs the rocket in different directions. This movement of the joystick will be mostly handled within this class.

3. SkyFlux.java

SkyFlux currently handles the main sky as well as the scrolling clouds. Perhaps in the future we will add more things to the sky and hence, this class will be required to handle it.

4. AsteroidFlux.java

In the future, we plan on adding asteroids which fly towards the rocket. We plan to require these asteroids to be destroyed by the user. The logic of the movement of these asteroids will be handled within this class.

RocketBlast-android

This project consists of one package. This package is as follows:

com.rocket.blast

This package currently contains one file. This file is as follows:

1. MainAvtivity.java

This class will launch our game when it is being deployed to an android device.

Within rocketBlast-android, we also have a folder structure as follows: assets>data,images>rocket,sky

Within rocket, we have placed our image of our rocket. Within sky, we have placed our main sky image, as well as our two cloud images. We have experimented a bit, so you will find a few other images in both of these folders.

com.rocket.blast

1. Main.java

This class will launch our game when it is being deployed on a desktop.
